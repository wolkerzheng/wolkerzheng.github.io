AI范式转变

一个时代正在翻篇

2026年3月，小米大模型负责人罗福莉接受张小珺的3.5小时深度访谈，抛出了一个判断：AI战争进入第二幕。

她认为，行业正从以预训练（Pre-train） 为主的"Chat时代"，全面转向以后训练（Post-train） 为主的"Agent时代"。这个转变的标志，是Anthropic推出的OpenClaw框架。这不是一次渐进式升级，而是底层范式的根本切换——竞争的核心赛点，从"谁的模型参数更大"，变成了"谁的Agent框架更强、RL规模化更狠、长上下文更稳"。

这期访谈信息密度极高，涵盖技术路线、竞争格局、组织管理、AGI预判等多个维度。以下是核心内容的系统梳理。

范式巨变：从Chat到Agent

时代切换的标志

罗福莉将2026年定义为分水岭。Chat时代的逻辑是"堆参数、刷Benchmark"，谁的基座模型大，谁就赢。而Agent时代的逻辑完全不同——核心赛点转移到Agent框架编排能力、强化学习（RL）规模化、长上下文（Long Context）、持久记忆、自主行动这五个维度。

这个转变的标志性事件是OpenClaw的出现。

OpenClaw：不是工具，是划时代框架

罗福莉对OpenClaw的评价极高，她个人试用后兴奋得睡不着，并称赞其有"灵魂感"。但她的评价并非停留在体验层面，而是给出了三个硬核判断：

模型能力越级。接入OpenClaw后，小米MiMo模型在85%的任务上可接近Claude Sonnet的水平。这意味着一个中层模型，通过精心编排的Agent框架，就能大幅弥补自身能力短板，接近顶流表现。

技术本质不是"Claude Code + IM"。OpenClaw的核心是系统编排——通过分层记忆体系、精细上下文管理等工程手段，让模型的实际输出远超其裸跑能力。这是用"框架设计"来弥补"模型能力"的路径，对国内模型追赶尤其有价值。

开源加速迭代。OpenClaw的开源特性让所有人都可以修改和适配，极大加快了Agent生态的迭代速度。罗福莉甚至一天花掉1000美元的Tokens来测试，称其为"改变世界的AI工具"。

Harness Engineering：新工种诞生

罗福莉提出了一个值得关注的概念——Harness Engineering。AI工程化的演进路径正在经历三阶段跃迁：

1. 写提示词（Prompt）：早期阶段，通过优化单次输入获得更好回答。 2. 管上下文（Context）：中期阶段，学会组织多轮对话的信息结构。 3. 设计稳定工作的系统（Harness）：当前阶段，核心目标变成让AI从聪明个体变成可靠工程系统。

这本质上是从"炼丹"到"工程"的转变。一个能稳定工作的Agent系统，需要考虑错误恢复、状态管理、工具编排、成本控制等一系列工程问题，这已经远远超出"写好Prompt"的范畴。

竞争格局：中美差距正在消失

预训练代差已基本抹平

罗福莉给出了一个大胆但具体的判断：中美在预训练上的代差已经缩至2-3个月。国内小米MiMo、Kimi等模型，与Claude Opus 4.6的差距仅在这个范围内。这意味着在预训练层面，大家基本站在同一起跑线上。

但她同时指出了一个门槛：1T参数基座是接近Claude 4.6水平的必要入场券。参数规模和上下文长度共同决定智能上限，而激活参数效率比总参数量更重要。

算力分配的逆转

罗福莉透露了一组关键数据——算力配置比例的逆转：

| 时代 | 研究 : 预训练 : 后训练 | |------|----------------------| | Chat时代 | 3 : 5 : 1 | | Agent时代 | 3 : 1 : 1 |

在Chat时代，预训练占据绝对主导地位（占比约55%）。而进入Agent时代，预训练和后训练的投入已经持平，研究投入反而成为最大的单项。这个数据直接印证了"后训练主导"的判断。

行业共识与路径判断

罗福莉认为Anthropic的路径是正确的——长上下文 + RL + 安全对齐。国内团队在这一点上已经形成共识，真正的胜负手在于：Agent拥抱速度、研究敏捷度、新范式落地能力。

换句话说，模型层面的差距正在快速缩小，真正的竞争壁垒将建立在谁能更快地把Agent能力转化为可用的产品和场景。

技术路线：MiMo-V2的架构选择

Hybrid Attention：不走主流路

MiMo-V2在架构上做了一个关键的差异化选择——放弃当时主流的MLA（全注意力），转而采用Hybrid Attention（混合注意力）架构。

具体实现是局部滑动窗口 + 全局注意力的混合方案，稀疏比达到7:1（全局:滑动窗口），比Flash的5:1更加极致。这个选择的核心考量是长文效率和成本——在Agent场景下，模型需要处理大量上下文信息，长文效率直接决定了Agent的实际可用性和经济可行性。

MTP：多Token预测的工程价值

MiMo还适配了MTP（Multi-Token Prediction，多Token预测）架构。MTP的核心价值不是提升模型智能，而是工程效率：

  • 不增加幻觉
  • 降低单token推理成本
  • 提升GPU利用率

当MTP的命中率高时，它成为推理加速的核心引擎，可带来1.5-2倍的推理速度提升。这对于Agent场景尤为重要——Agent需要频繁调用模型，推理速度和成本直接决定了系统的可行性。

Agent能力的真实检验标准

罗福莉提出了一个务实的判断标准：能否接入Claude Code/OpenClaw稳定工作。她强调Benchmark高分不等于真实场景好用，真正检验Agent能力的，是它在复杂任务中的稳定性和可靠性。

组织管理：无层级、无Deadline的平权团队

极致扁平的实验

罗福莉管理着一个约100人的MiMo团队（含数据、训练、推理、产品、多模态），核心迭代仅20-30人，实习生占比很高。但最引人注目的不是规模，而是组织方式：

无职级、无小组、无leader、无deadline、无固定分工。她一个人直接与100人沟通。

这套管理哲学的核心是平权有利于创新——在她看来，层级等于约束，约束压制创造力。人随项目自然流动，预训练人员转做后训练，反而带来了数据多样性的优势。

"每天和AI对话100次，否则辞职"

这是罗福莉对团队的要求。她的逻辑很简单：点燃团队对AI的直觉和热情，比任何管理手段都有效。

结果也验证了这个判断——团队被点燃后，3-4周就完成了以前可能需要30-40周的研究量，效率提升近10倍。

招人标准：好奇心 > 经验

罗福莉的用人标准很特别：不看大模型经验，招有好奇心、数据直觉、跨领域思考能力的人。她甚至偏爱大二大三的本科生，因为他们的想象力未被固化。很多人没有大模型背景，完全从零培养。

这个选择背后是一个判断：在范式快速切换的时期，学习能力和想象力比存量经验更有价值。

AGI时间表：2年内落地

罗福莉给出了明确的AGI时间表：

  • 当前：AGI已实现约20%
  • 2026年底：预计达到60%-70%，生产力开始爆发，大量工作被替代
  • 2年内：实现AGI，定义是"大部分人工作模式被颠覆，先工作、后生活"

关键节点是AI训AI——当模型能够自主做研究、自主迭代、自主提效时，就是AGI真正到来的标志。她认为届时效率提升可达10倍以上。

另一个值得关注的推论是：Agent普及后，推理芯片需求将暴涨5-10倍，低成本推理将成为核心命题。这也解释了为什么MiMo在架构上如此看重长上下文的效率——这是Agent时代的基础设施级需求。

开源与闭源的战略选择

罗福莉对这个问题的态度很鲜明：

开源的目的是加速AGI进程——分散算力负担、繁荣生态、推动芯片和框架的协同创新。

闭源的原因通常只有一个——缺乏不可替代的战略生态位，只能把模型本身当作唯一壁垒。

小米的选择是开源+生态。依托小米的硬件生态，走"模型 + 框架 + 终端"的协同路径，快速落地Agent应用。这不是纯粹的利他主义，而是基于自身生态优势的战略选择。

对研究者和个人的启示

访谈最后，罗福莉谈到了一个很多人不愿面对但必须面对的现实：

AI已经能做研究、写代码、设计实验、验证想法。创造力工作也会被Skill化和Workflow化。她认为，人类的核心价值将重新定义——AI替代的是无想象力的重复性工作，而想象、判断、创造、定义问题这些能力，才是人类真正的护城河。

2026年的主旋律，是高生产力场景的突破。AI正在从"写文案"升级为"做复杂任务、替代专业工作"。

一句话总结

罗福莉的核心判断可以浓缩为一句话：AI战争进入第二幕，Agent是唯一解；中国追上美国就在这两三个月；平权小团队加强研究投入是胜出关键；2年内AGI落地，重塑一切。

至于这个判断是对是错，时间会给我们答案。但有一点是确定的——范式已经切换，还用Chat时代的思维打Agent时代的仗，注定会被甩在后面。
