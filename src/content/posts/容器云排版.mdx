在内网离线环境的 Kubernetes 集群中部署你已打包的 m3e-onnx-image:latest 镜像，需要完成以下几个步骤：

1. 将镜像导入集群节点（或私有仓库）
方式一：直接将镜像加载到每个工作节点（不推荐）
如果你只有少量节点，可以手动将镜像 load 到每个节点上：

bash
docker load -i m3e-onnx-image.tar
然后设置 imagePullPolicy: IfNotPresent 或 Never，让 Kubelet 使用本地镜像。

方式二：搭建内网私有镜像仓库（推荐）
在内网部署一个私有镜像仓库（如 Harbor、Docker Registry），将镜像推送到仓库，然后所有节点统一从该仓库拉取。

bash
# 给镜像打上私有仓库地址的标签
docker tag m3e-onnx-image:latest <your-registry>/m3e-onnx:latest

# 推送到私有仓库
docker push <your-registry>/m3e-onnx:latest
之后在 Kubernetes 中可直接引用该地址。

2. 编写 Kubernetes 编排文件
假设你的服务监听容器内 8001 端口，需要对外提供访问。下面提供两个核心资源：Deployment 和 Service。

创建 deployment.yaml
yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: m3e-onnx-deployment
  namespace: default  # 可修改
spec:
  replicas: 1         # 按需调整
  selector:
    matchLabels:
      app: m3e-onnx
  template:
    metadata:
      labels:
        app: m3e-onnx
    spec:
      # 如果镜像已加载到每个节点且不推送到仓库，使用 imagePullPolicy: IfNotPresent
      # 如果推送到私有仓库，请使用 imagePullPolicy: Always 或 IfNotPresent
      containers:
      - name: m3e-onnx
        image: <your-registry>/m3e-onnx:latest   # 或 m3e-onnx-image:latest
        imagePullPolicy: IfNotPresent            # 根据情况调整
        ports:
        - containerPort: 8001                    # 容器内服务端口
          name: http
        resources:
          requests:
            memory: "512Mi"
            cpu: "500m"
          limits:
            memory: "1Gi"
            cpu: "1000m"
        # 如果需要GPU，请添加以下配置（需集群支持）
        # resources:
        #   limits:
        #     nvidia.com/gpu: 1
        # 健康检查（根据实际应用调整）
        livenessProbe:
          httpGet:
            path: /health   # 替换为实际健康检查路径
            port: 8001
          initialDelaySeconds: 30
          periodSeconds: 10
        readinessProbe:
          httpGet:
            path: /ready
            port: 8001
          initialDelaySeconds: 5
          periodSeconds: 5
创建 service.yaml
yaml
apiVersion: v1
kind: Service
metadata:
  name: m3e-onnx-service
  namespace: default
spec:
  selector:
    app: m3e-onnx
  ports:
  - protocol: TCP
    port: 8000          # Service 对外暴露的端口
    targetPort: 8001    # 容器内的端口
  type: ClusterIP       # 集群内部访问，如需外部访问改为 NodePort 或 LoadBalancer
如果需要外部访问，可使用 NodePort
yaml
type: NodePort
然后通过 <node-ip>:<node-port> 访问。或者使用 Ingress 提供 HTTP 路由。

3. 应用到 Kubernetes 集群
将上述 YAML 文件保存后，使用 kubectl 创建资源：

bash
kubectl apply -f deployment.yaml
kubectl apply -f service.yaml
验证部署：

bash
kubectl get pods
kubectl logs <pod-name>
kubectl get svc
4. 测试服务
集群内部测试：通过 Service 的 ClusterIP 访问

bash
kubectl run -it --rm test --image=busybox -- wget -qO- http://m3e-onnx-service:8000
外部访问（NodePort）：获取节点 IP 和端口

bash
kubectl get svc m3e-onnx-service
然后在浏览器访问 http://<node-ip>:<node-port>。

5. 可选：配置存储、环境变量等
如果你的应用需要持久化数据或环境变量，可在 Deployment 中添加：

yaml
env:
- name: MODEL_PATH
  value: "/models/m3e.onnx"
volumeMounts:
- name: model-storage
  mountPath: /models
volumes:
- name: model-storage
  hostPath:
    path: /data/models   # 内网节点上的路径
注意事项
镜像拉取问题：离线环境中如果镜像未推送到私有仓库，务必在每个节点手动 docker load，并设置 imagePullPolicy: IfNotPresent。

私有仓库认证：如果私有仓库需要认证，需创建 imagePullSecrets。

资源限制：根据模型加载所需内存/显存调整 limits，防止 Pod 被驱逐。

健康检查：确保 /health 或 /ready 路径存在，否则探针失败会导致 Pod 重启。

按照上述步骤，你就能在内网 Kubernetes 中成功编排并运行该镜像。如果有具体错误或需要调整，可提供更多信息以便进一步协助。
